# 端到端验证清单

部署后按本清单逐项验证。目标：确认三期功能（业务主线 + 财务 + 多租户隔离）在真机上端到端跑通。

> 前置：`./update.sh` 部署完成，`docker compose ps` 三个容器均为 Up；`docker compose logs -f backend` 看到 `Started ConsumablesApplication` 且无 Flyway 报错。

---

## 阶段 0：启动与建表

- [ ] `docker compose ps`：mysql / backend / frontend 均 Up（mysql 为 healthy）
- [ ] 后端日志无 `FlywayException`，V1/V2/V3 迁移成功
- [ ] 后端日志出现"已初始化平台管理员账号: admin"
- [ ] 浏览器访问 `http://<公网IP>:8090` 自动跳转到 `/login` 登录页

---

## 阶段 1：认证与安全

- [ ] 用错误密码登录 `admin` → 提示"账号或密码错误"
- [ ] 不带 token 直接访问业务接口（如浏览器开发者工具里请求 `/api/goods/page`）→ 返回 401 未认证
- [ ] 用 `admin` / `admin123` 登录成功 → 进入"商家管理"界面（不显示业务菜单）
- [ ] 点击"退出登录" → 回到登录页，本地 token 被清除

> 若已改默认密码，用你设置的密码。生产环境务必确认已改 `admin123` 与 `jwt.secret`。

## 出现问题时排查

| 现象 | 排查方向 |
|------|----------|
| 登录页刷不出 | 前端容器日志、8090 端口安全组、Nginx 配置 |
| 登录报 401/500 | 后端日志、数据库连接、admin 账号是否初始化 |
| 商家能看到别家数据 | **立即停用对外访问**，检查租户拦截器是否生效（后端日志 SQL 是否带 S_TENANT_ID 条件）|
| 库存/金额数字不对 | 检查单位换算、金额分/元换算 |
| Flyway 报错 | 看是否 V1/V2/V3 顺序执行；无存量数据可考虑清库重来 |

> 阶段 5（隔离）是 SaaS 平台的安全底线，务必重点验证。任一项发现能看到别家数据，应立即停止对外提供服务并排查。
